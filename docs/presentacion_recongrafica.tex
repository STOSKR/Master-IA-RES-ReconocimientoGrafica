\begin{frame}{No es solo OCR}
  \Large
  El OCR lee texto, pero la serie sale de la imagen completa.

  \vspace{0.5cm}
  \begin{columns}
    \column{0.33\textwidth}
    \textbf{\textcolor{green}{Visión}}\\
    Detecta layout, ejes y línea.

    \column{0.33\textwidth}
    \textbf{\textcolor{cyan}{OCR}}\\
    Lee fechas y precios.

    \column{0.33\textwidth}
    \textbf{\textcolor{yellow}{Geometría}}\\
    Convierte píxeles en datos.
  \end{columns}
\end{frame}

\begin{frame}{Pipeline}
  \Large
  \begin{enumerate}
    \item Segmentación del layout.
    \item OCR de anclas en ejes.
    \item Extracción de la señal visual.
    \item Calibración píxel-dato.
    \item Exportación y depuración.
  \end{enumerate}
\end{frame}

\begin{frame}{Reconstrucción geométrica}
  \Large
  Se ajustan dos transformaciones lineales:

  \vspace{0.4cm}
  \begin{block}{Eje X}
    \texttt{pixel\_x $\rightarrow$ fecha}
  \end{block}

  \begin{block}{Eje Y}
    \texttt{pixel\_y $\rightarrow$ precio}
  \end{block}

  \vspace{0.3cm}
  Las anclas OCR definen la escala real de la gráfica.
\end{frame}

\begin{frame}{Batch automático}
  \Large
  El comando procesa todas las imágenes de \texttt{data/raw}.

  \vspace{0.4cm}
  \begin{block}{Comando}
    \texttt{python -m recongrafica batch --raw-dir data/raw --out-dir outputs/auto --use-ocr}
  \end{block}

  \vspace{0.2cm}
  Por cada imagen genera: \textbf{CSV, JSON, debug PNG y mask PNG}.
\end{frame}

\begin{frame}{Problemas encontrados}
  \Large
  \begin{itemize}
    \item ROI del eje X demasiado baja.
    \item Máscara inicial demasiado gruesa.
    \item OCR con falsas anclas.
    \item Fechas parciales como \texttt{27/05/20}.
  \end{itemize}

  \vspace{0.5cm}
  El debug visual permitió localizar y corregir estos fallos.
\end{frame}
